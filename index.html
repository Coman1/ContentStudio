What these demos cover
🔗 Ingestion layer
Connects to SharePoint (Modern and Classic), PDF, and Word documents. Incremental sync, metadata extraction, pipeline status visible to non-technical users, and a manual upload interface.
📁 Knowledge store
Single source of truth with version history, diff view, rollback, domain segregation (Pensions / Wealth), and sensitivity classification (Internal / Restricted / Highly Restricted).
✏️ Curation studio
Knowledge gap queue from agent failures, automated conflict detection between files, freshness scoring, AI writing assistant, and Consumer Duty readability scoring inline in the editor.
🎓 Training builder
Generate training modules and assessments from knowledge files. Auto-sync when source knowledge changes. Certification pathways for handlers and advisers. Learning analytics.
🔬 Agent lab
No-code prompt playground, retrieval inspector, A/B knowledge testing, agent configuration, and guardrail builder — all without engineering involvement.
📊 Arize AX integration
LLM tracing with span waterfall, hallucination detection, guardrail trip alerts, experiment tracking, production monitors, and annotation queue for human review.
👥 Reviews & sign-off
Structured review workflows, SME portal accessible via email link (no platform account needed), compliance sign-off trail exportable for FCA audit.
🛡️ Governance
Role-based access control with Entra ID SSO, immutable audit log, sensitivity tiers, 7-year retention, and publish gates linked to evaluation results.

Important notes

These are concept demos, not production software. All names, data, and scenarios are fictional. No real customer, colleague, or Aviva data is used anywhere.

The demos run entirely in your browser — no server, no backend, no data sent anywhere. All interactions are simulated.

They illustrate the requirements set out in the Knowledge Ingestion Tool MVP Requirements v0.2 and Content Studio MVP Requirements v0.1 documents. Technology choices for the actual build are defined separately.
